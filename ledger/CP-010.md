# CP-010 — Strategy Engine

**Date:** 2026-04-19
**Goal:** Five strategy detectors — VWAP Reclaim, Opening Range Breakout, Pullback Continuation, Mean Reversion, Failed Breakout — each returning Entry / Stop / Target / Confidence / Notes. Ride on the CP-008 candle stream and CP-009 engine pattern.

## Delivered

### New package `backend/app/strategies/`

```
strategies/
├── types.py         # StrategyDraft (dataclass) + StrategySignal (Pydantic)
├── indicators.py    # rolling VWAP, SMA, EMA, stddev, ATR
├── detectors.py     # 5 pure detector functions + ALL_STRATEGIES tuple
└── engine.py        # StrategyEngine — 50-bar buffer + 200-signal history
```

### Types (`types.py`)
- `StrategyName` literal: `vwap_reclaim | opening_range_breakout | pullback_continuation | mean_reversion | failed_breakout`.
- `StrategyDirection = "LONG" | "SHORT"`.
- `StrategyDraft` — dataclass returned by detectors (no validation overhead at bar-close cadence).
- `StrategySignal` — Pydantic model the engine produces, adding `symbol / timeframe / time / risk_reward`.

### Indicators (`indicators.py`)
Pure, dependency-free helpers: `typical_price`, `rolling_vwap`, `sma`, `ema` (full series), `stddev`, `atr`. Each returns `None` when the window is too short — detectors guard on that rather than catching.

### Detectors (`detectors.py`)
Each takes a candle buffer (most recent last) and returns a `StrategyDraft | None`. All five handle **long and short** variants. Confidence bands 55–95; invalid geometry (target on wrong side of entry, zero risk) rejected internally so the engine always gets a well-formed draft.

| Strategy                    | Trigger                                                                                             | Stop                      | Target           |
| --------------------------- | --------------------------------------------------------------------------------------------------- | ------------------------- | ---------------- |
| `vwap_reclaim`              | Prev close on one side of rolling 20-bar VWAP, current close crosses through                        | VWAP ± ATR(14)            | Entry ± 2R       |
| `opening_range_breakout`    | First 3 bars define the range; first close outside fires                                            | Opposite edge of range    | Entry ± OR range |
| `pullback_continuation`     | EMA(20) rising 3 bars + price above, 3 lower-high pullback bars, then bullish reclaim of prev high   | Pullback low              | Entry + 2R       |
| `mean_reversion`            | Close ≥ 2σ outside 20-bar mean (±2σ band)                                                          | Beyond extreme by 0.5σ    | 20-bar mean      |
| `failed_breakout`           | Prev bar poked outside prior 10-bar H/L, current bar closes back inside against the break direction | Prev bar's extreme        | Opposite edge    |

"Opening range" for the mock/demo build means the first three bars of the engine's rolling buffer rather than a session-aware window — documented in the module docstring; drop-in replacement when the provider supplies session metadata.

### Engine (`engine.py`)
- Per-`(symbol, timeframe)` candle buffer sized 50 (largest detector needs ~22 bars; room to spare).
- `on_closed_candle(symbol, timeframe, candle) → list[StrategySignal]` runs every detector, computes `risk_reward = |target − entry| / |entry − stop|`, clamps confidence to [0, 100], appends to a 200-entry history.
- `recent(symbol, timeframe, limit)` serves the REST endpoint.

### Service integration (`market_data/service.py`)
`MarketDataService` now accepts an optional `strategy_engine: StrategyEngine`. The aggregation loop fires the engine on each `update.closed` and publishes signals on the **same** `StreamHub` with:

```json
{
  "type": "strategy.signal",
  "symbol": "AAPL",
  "timeframe": "1h",
  "signal": {
    "symbol": "AAPL", "timeframe": "1h",
    "strategy": "vwap_reclaim", "direction": "LONG",
    "entry": 101.5, "stop": 98.85, "target": 106.81,
    "risk_reward": 2.0, "confidence": 77,
    "notes": "Price reclaimed rolling VWAP at 100.03 with bullish close.",
    "time": 1776646020
  }
}
```

A single WS subscription to `(symbol, timeframe)` now delivers three event kinds: `candle.update` / `candle.closed`, `pattern.detected`, and `strategy.signal`.

### API
- `GET /api/v1/strategies/{symbol}?timeframe=1h&limit=50` → `StrategiesResponse { symbol, timeframe, signals: StrategySignal[] }`.
- Wired in `api/v1/router.py`. App lifespan now builds `MarketDataService(..., strategy_engine=StrategyEngine())`.

## Verification

**Seeded detector walkthrough** — hand-fed one deterministic buffer per strategy:

```
1. vwap_reclaim (long)           LONG  entry=101.50 stop=98.85  target=106.81 conf=77
   notes: Price reclaimed rolling VWAP at 100.03 with bullish close.
2. opening_range_breakout        LONG  entry=102.50 stop=99.00  target=104.50 conf=85
   notes: Price broke above opening range (99.00–101.00).
3. pullback_continuation (long)  LONG  entry=106.15 stop=105.10 target=108.25 conf=76
   notes: Uptrend pullback resolved with bullish close above prior bar high.
4. mean_reversion (long)         LONG  entry=95.00  stop=93.44  target=99.76  conf=85
   notes: Close is 4.3σ below 20-bar mean 99.76; fade toward mean.
5. failed_breakout (short)       SHORT entry=100.80 stop=103.50 target=99.50  conf=85
   notes: Failed upside breakout above 101.50; closed back in range.
```

R:R sanity: VWAP reclaim and pullback both size at 2R by construction; ORB targets the range itself; mean reversion targets the mean; failed breakout targets the opposite edge.

## Decisions & notes

- **"Opening range" = first 3 buffer bars**, not a session-aware 9:30-to-10:00 window. Mock provider has no session metadata and we don't want to block CP-010 on that. The module docstring calls this out; real providers in a later integration will supply session marks so ORB can use them.
- **Rolling VWAP, not session VWAP.** Same reasoning — we compute VWAP over the last 20 buffered bars. Preserves the crossing-semantics without requiring session boundaries.
- **Detectors reject invalid geometry themselves** (`_valid_long` / `_valid_short`) so the engine can trust every draft. Simpler than centralizing sanity in the engine.
- **Signal history capped at 200.** A backtested strategy mining historical sessions will want a bigger buffer; for the live stream 200 covers several days at any active timeframe.
- **Same hub, `type: "strategy.signal"`.** Matches CP-009's pattern-event design. One WebSocket subscription per (symbol, timeframe) delivers candles, patterns, and strategies together — which is what the frontend actually consumes.
- **Confidence ranges kept in 55–95.** Low-confidence signals add noise without marginal value; we don't emit near-threshold cases. Calibrating against backtested win-rates is a CP-011 concern.

## Not in scope (deferred)

- Session-aware ORB, session VWAP.
- Backtested confidence calibration (probability vs. shape-fit score).
- Persistence — engine state is in-process only.
- Frontend consumption of `strategy.signal` events (list, chart overlay, alerts hook-up).
- Strategy-specific parameters exposed via config (ATR period, band width, pullback lookback).
