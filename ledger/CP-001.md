## Delivered

```

├── docs/           Architecture and developer docs

```

### Frontend
- `web/` renamed to `frontend/` via `git mv` — commit history preserved.
- No code changes in this sprint; just relocated. Build verified from new path.

### Backend (`backend/app/`)
- **`main.py`** — FastAPI app factory. Mounts CORS middleware, `/health`, and an empty `/api/v1` router that feature sprints will populate.
- **`api/health.py`** — `GET /health` returns `{status, service, version, environment}`.
- **`api/v1/router.py`** — APIRouter with `/v1` prefix, mounted at `/api`. No routes yet — this is the composition root future sprints plug into.
- **`core/config.py`** — `Settings` class via `pydantic-settings`. Reads `backend/.env`. All CP-008+ values (DATABASE_URL, MARKET_DATA_*) are optional for now.
- **`db/session.py`** — Lazy async SQLAlchemy engine + session factory. Doesn't connect at import time; raises with a clear error if `DATABASE_URL` is unset and something tries to use it.

### Dependencies
- Backend: `fastapi`, `uvicorn[standard]`, `pydantic`, `pydantic-settings`, `httpx`, `sqlalchemy[asyncio]`, `asyncpg`, `alembic`, `websockets`. Dev: `ruff`, `pytest`, `pytest-asyncio`.
- Python ≥ 3.11, managed with `uv` (fallback: pip in a venv).

### Shared config
- `backend/.env.example`, `frontend/.env.example`, and root `.env.example` as an aggregate inventory.
- Each service reads its own file. Root one is documentation.

## Decisions & notes
- Chose `uv` over poetry/hatch for speed and minimal config. `pyproject.toml` is standard PEP 621 — any resolver works.
- **No placeholder feature files.** `api/v1/router.py` is empty on purpose; feature sprints will add real routers, not stubs. Same reason there's no `candles.py` or `ws.py` yet — they come with CP-008.
- Database session is **lazy**. The backend boots without a DB so CP-002/003 frontend work isn't blocked on infra.
