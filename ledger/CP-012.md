# CP-012 — Browser Companion

**Date:** 2026-04-19
**Goal:** Chrome extension that detects the symbol on the current page, shows quick signals/patterns from the backend, and opens the full app for deep dives.

## Delivered

### New top-level `extension/` directory

```
extension/
├── manifest.json           # MV3
└── popup/
    ├── popup.html          # panel UI
    ├── popup.css           # dark premium styling (no external deps)
    ├── popup.js            # detection + fetch + render
    ├── options.html        # settings page
    └── options.js          # api/app base + default timeframe
README.md                   # how to load-unpacked + supported sites
```

No build step, no bundler, no icons. Pure HTML/CSS/JS served directly by Chrome. (Icons intentionally deferred — Chrome's default puzzle-piece shows up for now.)

### Manifest (`manifest.json`)
- Manifest V3.
- `action.default_popup` → `popup/popup.html`.
- Permissions: `activeTab` (read current tab's URL), `storage` (sync settings).
- `host_permissions`: localhost backend/frontend for dev + Yahoo Finance, TradingView, Google Finance, Schwab so the popup can `fetch` without being blocked.
- `options_ui` registers `popup/options.html` for per-user config.

### Symbol detection (`popup.js`)
URL-only — no content scripts injected into third-party pages. A ranked list of per-site rules + a generic `?symbol=` fallback:

| Source           | Rule                                                       |
| ---------------- | ---------------------------------------------------------- |
| Yahoo Finance    | `/^\/quote\/([A-Z0-9\.\-=\^]+)/i` on `finance.yahoo.com`   |
| TradingView      | `/symbols/EXCHANGE-SYM/` or `?symbol=EXCHANGE:SYM`         |
| Google Finance   | `/^\/finance\/quote\/([A-Z0-9\.\-]+)/i`                    |
| Schwab           | `?symbol=` or `?sym=` on any `*.schwab.com` URL            |
| Generic fallback | `?symbol=` on any host (1–10 ticker-safe chars)            |

The extraction helper is exposed on `globalThis.__extractSymbolFromUrl` so it can be smoke-tested in Node.

### Popup UI (`popup.html` + `popup.css`)
Matches the frontend's dark premium aesthetic:
- Top bar — logo mark + "Open app ↗" button that deep-links to `/symbol/{ticker}`.
- Symbol card — detected ticker, source site label + URL, timeframe picker (1m / 5m / 15m / 1h / 4h / 1d).
- Recent signals — latest four entries from `/api/v1/strategies/{symbol}` with direction colour, entry/target/stop, confidence pill, notes.
- Recent patterns — latest four entries from `/api/v1/patterns/{symbol}` with direction colour + confidence.
- No-symbol state with an explainer of supported sites.
- Footer — status line + settings link.

All network data is injected via `textContent` / `setAttribute` — never `innerHTML` — so malicious or unexpected JSON can't introduce markup.

### Options page (`options.html` + `options.js`)
Form with three fields, persisted in `chrome.storage.sync`:
- Backend API base (default `http://localhost:8000`).
- Frontend app base (default `http://localhost:3000`).
- Default timeframe.

### Backend change
`backend/app/main.py` now adds `allow_origin_regex=r"chrome-extension://.*"` to the CORS middleware. Any installed extension id can hit the API without the user editing `FRONTEND_ORIGINS`.

## Decisions & notes

- **URL-only detection, no content scripts.** Injecting scripts into Yahoo / Schwab / TradingView raises the review bar with Chrome Web Store and adds potential privacy concerns. URL inspection covers the common case; sites that hide the ticker behind app routing (notably Schwab's in-app trade view) aren't fully supported — documented as a limitation.
- **No icons yet.** Chrome falls back to the generic puzzle-piece toolbar icon. Adding `icons/{16,48,128}.png` and referencing them in the manifest is strictly a pre-publish polish step; omitting them keeps this sprint free of binary assets.
- **Pure vanilla, no bundler.** Extension is three HTML/CSS/JS files. Adding Vite or webpack buys nothing for this surface and would add a build step that has to stay in sync with the frontend.
- **CORS regex, not env list.** Per-extension-id allowlisting would mean users editing `backend/.env` after every `chrome://extensions` re-install. `chrome-extension://*` keeps local dev frictionless. For production, tighten to a specific published extension id.
- **Polling, not WebSocket.** Popup fetches `/api/v1/strategies` and `/api/v1/patterns` on open and on timeframe change. A WS would keep data current after the popup opens, but popups close on blur anyway — polling-on-open is simpler and cheaper.

## Not in scope (deferred)

- Icons (PNG assets).
- Content-script symbol detection for Schwab's in-app routes (needs DOM scraping, which requires broader host permissions).
- Live-streaming via WS from inside the popup.
- Firefox / Safari / Edge companion builds (MV3 is compatible; branding and manifests would need variants).
- Extension Web Store publish flow.
