## Decisions & notes
- **Kept `ScannerTimeframe` scanner-local** rather than importing `Timeframe` from `lib/types.ts`. Filter values are lowercase (`"1h"`) while the generic `Timeframe` uses mixed case (`"1H"`) for chart-library compatibility. Unifying is a future cleanup if the same display labels are needed elsewhere.
- **Did not extract `TrendBadge` / `ConfidenceBar` to `components/common/`.** Each page uses a visually distinct pill/bar per Figma; sharing would flatten the design. Same reasoning as CP-002.
- **Filter composition intentionally AND.** All active filters must match. If we find users want OR semantics for sentiment, that's a CP-008+ change.
