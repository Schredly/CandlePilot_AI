# CP-005 ŌĆö Symbol Detail Build

**Date:** 2026-04-19
**Goal:** Symbol detail page with a TradingView Lightweight Charts candlestick chart, volume bars, pattern markers, signal card with confidence + entry/stop/target, AI explanation panel, and notes area.

## Delivered

### Chart engine: Recharts ŌåÆ Lightweight Charts
The original Figma port used Recharts with a range-`dataKey` hack to approximate candles on top of a bar chart. CP-005's real deliverable was swapping in the proper library:

- Installed `lightweight-charts@5.1.0`.
- Rewrote `components/symbol/CandlestickChart.tsx` from scratch as a client component that:
  - Creates the chart once on mount (`createChart` + `autoSize`) and cleans up on unmount.
  - Adds a `CandlestickSeries` with up/down colors matching the design system (`#10b981` / `#ef4444`).
  - Adds a `HistogramSeries` for volume pinned to the bottom 20% of the chart via a dedicated price scale.
  - Renders pattern markers on-chart via `createSeriesMarkers` (v5 plugin API). Previously markers existed only as chips below the chart.
  - Renders support/resistance as dashed price lines via `series.createPriceLine` with axis labels.
  - Four discrete effects ŌĆö init, data sync, markers sync, levels sync ŌĆö so props-driven updates don't tear down the chart.
- Removed the old Recharts-based `CustomCandle` shape, `ChartTooltip` (Lightweight Charts has its own crosshair/legend), and the `[low, high]` range-dataKey trick.

### Data model
- `Candle.time` changed from `string` to `number` (Unix seconds) ŌĆö the native format Lightweight Charts accepts and what a real market-data feed will emit. Updated `lib/types.ts` + `lib/mock-candles.ts`.
- Mock candles now emit contiguous **hourly** bars ending snapped to the current hour (was 60 bars starting at a fixed 2024 date). The axis shows recent-looking times instead of stale historical labels.
- Bumped mock length from 60 ŌåÆ 120 bars so the chart has room to pan/zoom.
- New file `lib/mock-symbol.ts` with `SymbolDetailData` + helper `getMockSymbolDetail(ticker)` bundling: candles, signal card, timeframe alignment, AI explanation, recent alerts, pattern markers (anchored to real bar timestamps, not array indices), and support/resistance levels.

### Page wiring
- `app/symbol/[ticker]/page.tsx` now calls `getMockSymbolDetail()` and spreads the result. One entry point instead of six inline mock constants.
- `IntelligencePanel` re-exports its types from `lib/mock-symbol.ts` to eliminate the duplicate type definitions.
- `TradeJournal` unchanged ŌĆö already prop-driven (`defaultNotes`).

### Other brief items
- Current signal card, confidence score, entry/stop/target ŌĆö already built (`IntelligencePanel` signal section). Unchanged.
- AI explanation panel ŌĆö already built. Unchanged.
- Notes area ŌĆö already built (`TradeJournal`). Unchanged.

## Decisions & notes
- **Pattern markers anchored by timestamp, not array index.** Previously `patterns: [{index: 45, type}]` ŌĆö fragile against any data refresh that reorders or trims bars. Now `patterns: [{time: 1746122400, type, color}]`. Maps directly to what CP-009's pattern engine will emit.
- **Four useEffects, not one.** Kept chart init separate from data/markers/levels sync so re-renders with new props update in place instead of recreating the chart.
- **Removed custom tooltip.** Lightweight Charts provides a built-in crosshair that tracks OHLCV cleanly. Trying to layer a React tooltip on top of a canvas chart is finicky.
- **Kept the pattern chip row below the chart.** Redundant with the on-chart markers but useful when patterns are off-screen after panning. Low cost, adds context.
