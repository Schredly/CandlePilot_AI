# CP-011 — Backtesting Lab

**Date:** 2026-04-19
**Goal:** Replay historical candles through the CP-010 detectors, simulate trades, and compute win rate / avg return / drawdown metrics plus the equity curve needed to power the Figma results charts.

## Delivered

### New package `backend/app/backtest/`

```
backtest/
├── types.py        # BacktestRequest, Trade, EquityPoint, BacktestMetrics, BacktestResult
├── metrics.py      # compute_metrics(trades, equity, initial, final)
└── runner.py       # run_backtest(service, request) — pure simulation over historical candles
```

### Types (`types.py`)
- `BacktestRequest(symbol, timeframe, strategy, bars, initial_balance, risk_per_trade)`. `strategy` is one of the 5 strategy names or `"all"`. `bars` clamped to [50, 5000]. `risk_per_trade` clamped to (0, 0.1].
- `Trade(strategy, direction, entry, stop, target, exit, entry_time, exit_time, outcome, return_pct, pnl)`. `outcome ∈ {win, loss, timeout}` (`timeout` = position still open at end of replay, closed at last bar's close).
- `EquityPoint(time, balance)` for the chart.
- `BacktestMetrics(total_trades, wins, losses, timeouts, win_rate, avg_return_pct, total_return_pct, total_pnl, max_drawdown_pct, sharpe, final_balance)`.
- `BacktestResult` bundles request + trades + equity_curve + metrics.

### Metrics (`metrics.py`)
Pure functions, no imports outside the types module:
- `win_rate = wins / total_trades`.
- `avg_return_pct` = arithmetic mean of per-trade `return_pct`.
- `total_return_pct` = `(final_balance / initial_balance − 1) × 100`.
- `max_drawdown_pct` = largest peak-to-trough decline in the equity curve.
- `sharpe` = `mean / stdev` of per-trade returns. **No annualization** — per-trade returns don't cleanly map to a trading calendar. Documented in the schema description.

### Runner (`runner.py`)
Feeds historical candles one-at-a-time through the selected detector(s) using the same function-based detectors CP-010 emits live. Simulation model (every assumption spelled out in the module docstring):

- **One open position at a time.** A signal that fires while a trade is open is skipped — pyramiding sizing is out of scope.
- **Entry at signal price**, which equals the signal bar's close. Accepts a small look-ahead (real fills happen on next bar's open) to keep simulation self-contained.
- **Exits on every subsequent bar** — stop triggers if the bar's low crosses through (long) or high crosses (short); target mirrors it.
- **Same-bar collision = stop wins.** Pessimistic assumption; standard for simple backtests and protects against overstating results.
- **Position size = `balance × risk_per_trade / |entry − stop|`.** One stop-out loses exactly `risk_per_trade` percent of current balance at that moment. Matches how traders actually size.
- **Open position at replay end** is closed at last bar's close and tagged `timeout`.

Historical candles come from `service.backfill()` which delegates to `provider.historical_candles()`. The MockProvider returns deterministic-by-symbol synthetic history, so a given request always produces the same metrics.

### API
- `POST /api/v1/backtest` — body: `BacktestRequest` JSON; response: `BacktestResult`.
- Wired in `api/v1/router.py`.
- No app lifespan changes; runner uses the existing `MarketDataService`.

Ran all five strategies + the combined "all" run against AAPL/1h over 500 bars, initial $10,000, 1% risk per trade:

Sanity checks that pass:
- **ORB fires once** (only the first close outside the opening range counts). ✓
- **Mean reversion on random-walk data loses money** (no mean to revert to). ✓ -14.41%.
- **Pullback continuation tops the list** — a trend-following strategy on a slightly-drifting random walk that by construction has upward bias. ✓.
- **`all` result ≈ weighted mix of the individual runs**, dragged down by mean_reversion. Trade count doesn't equal sum(individuals) because one-position-at-a-time skips overlapping signals. ✓.
- **Trade geometry is well-formed.** Every trade has `stop < entry < target` (long) or `target < entry < stop` (short); exit price equals stop or target on win/loss, last-bar close on timeout.
- **Equity curve has a point per trade + initial seed**, matching the frontend's chart expectations.

## Decisions & notes

- **`BacktestResult` shape matches the frontend's Figma chart.** `equity_curve` is `{time, balance}[]`, and `trades` carries everything the existing `TradeHistoryTable` already renders (`date`, `symbol`, `side`, `entry`, `exit`, `return`, `pnl`). Wiring the UI to the backend is deferred but the types line up — no refactor later.
- **Deterministic results from MockProvider.** Each symbol seeds a stable random walk; `POST /backtest` for the same symbol/bars always returns the same trade sequence. Lets the frontend cache results and users compare runs meaningfully. Real-provider historicals will obviously vary.
- **Sharpe left un-annualized.** Per-trade returns have no inherent cadence (a trade lasting 1 bar vs 50 bars skews any annualization). Callers that want a time-adjusted score can divide by sqrt(trades/year) themselves. The unitless signal-to-noise still serves the UI.
- **Fixed-risk sizing, not fixed-notional.** Prevents a big-stop trade from dwarfing the equity curve. Mirrors how real discretionary traders think.
- **Pessimistic same-bar resolution.** If both stop and target sit inside one bar's high-low range, we assume stop hit first. The opposite assumption would make ORB look unrealistically good on strong-breakout bars.
- **No fees/slippage.** Left out so the strategy-engine quality is legible. A `fees: float` parameter slots in trivially in a CP-014 hardening pass.

## Not in scope (deferred)

- Next-bar-open fills (look-ahead removal) and slippage/fee modeling.
- Multi-position sizing, pyramiding, scaling in/out.
- Intrabar tick simulation — we only see OHLC, not order of extremes.
- Wiring the Figma Backtesting page to this endpoint. The UI exists; swapping `lib/mock-backtesting.ts` for a `lib/api/backtest.ts` that calls `POST /api/v1/backtest` is a small focused task.
- Persisting backtest runs to Postgres (current results are request/response only).
- Walk-forward and Monte-Carlo permutations.
