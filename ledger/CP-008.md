# CP-008 — Real-Time Data Engine

**Date:** 2026-04-19
**Goal:** Backend market-data layer: live symbol streaming, candle generation across multiple timeframes, WebSocket feed to the frontend, reconnect logic, and a clean service abstraction — starting with one provider.

## Delivered

### Backend (`backend/app/market_data/`)

```
market_data/
├── types.py           # Tick, Candle, Timeframe, TIMEFRAME_SECONDS
├── provider.py        # MarketDataProvider (ABC)
├── providers/
│   └── mock.py        # MockProvider — deterministic synthetic feed
├── aggregator.py      # CandleAggregator — tick → multi-timeframe OHLCV
├── hub.py             # StreamHub — in-memory pub/sub for clients
└── service.py         # MarketDataService — orchestrates + reconnect
```

- **`types.py`** — `Tick`, `Candle` (Pydantic, Unix-seconds time), `Timeframe` literal (`1m`/`5m`/`15m`/`1h`/`4h`/`1d`), and a `TIMEFRAME_SECONDS` table.
- **`provider.py`** — single interface (`connect`, `disconnect`, `subscribe`, `unsubscribe`, `ticks()`, `historical_candles()`). Every future adapter (Alpaca/Polygon/Finnhub) implements this. Service layer never imports a vendor SDK.
- **`providers/mock.py`** — real, working provider. Random-walk ticks with deterministic per-symbol seeds. Emits one tick per subscribed symbol every 0.5s via an internal asyncio.Queue. `historical_candles` generates a matching synthetic backfill so the REST preload and the live stream line up visually.
- **`aggregator.py`** — bucket each tick by `time // tf_seconds` for every timeframe. Emits `CandleUpdate(timeframe, candle, closed)` whenever a bar advances — an in-progress update for the current bar, a `closed=True` event the moment a new bar begins.
- **`hub.py`** — `StreamHub.subscribe(symbol, tf) → asyncio.Queue`, `publish(...)`, `unsubscribe(...)`. Bounded queues (100 events) with drop-on-full policy — one slow client cannot stall the aggregator.
- **`service.py`** — `MarketDataService.start()` connects the provider and spins up a background task that drains `provider.ticks()`, feeds the aggregator, and publishes to the hub. The outer `while self._running` loop catches any non-cancellation exception, logs it, waits `reconnect_seconds`, and calls `provider.disconnect()` + `provider.connect()` again. This is the reconnect-logic surface — each concrete provider focuses on detecting failure (raising on connection drop) and the service handles recovery uniformly.

### API (`backend/app/api/v1/` + `main.py`)

- **`candles.py`** — `GET /api/v1/candles/{symbol}?timeframe=1h&limit=120` returns a `CandleSeriesResponse`. Validates the timeframe query against the `Timeframe` literal; rejects unknown values with 400.
- **`ws.py`** — `/ws` WebSocket with a simple JSON protocol:

  ```
  → { action: "subscribe",   symbol, timeframe }
  ← { type:  "subscribed",   symbol, timeframe }
  ← { type:  "candle.update" | "candle.closed", symbol, timeframe, candle }

  → { action: "unsubscribe", symbol, timeframe }
  ← { type:  "unsubscribed", symbol, timeframe }

  → { action: "ping" }
  ← { type:  "pong" }
  ```

  Each subscription gets its own relay task pumping its Queue to the socket. On disconnect every subscription is unsubscribed from the service so the provider drops any symbol with no remaining listeners.
- **`main.py`** — FastAPI `lifespan` builds the provider via `_build_provider()` (reads `MARKET_DATA_PROVIDER` env; defaults to `mock`), instantiates `MarketDataService`, stores it on `app.state.market_data`, and stops it cleanly on shutdown. `/ws` is mounted at the app root (no `/api/v1` prefix) so it matches `NEXT_PUBLIC_WS_URL=ws://host/ws`.

### Frontend

- **`frontend/src/lib/realtime/use-live-candles.ts`** — client hook: opens the WebSocket (auto-resolves protocol from `window.location`, overridable via `NEXT_PUBLIC_WS_URL`), sends `subscribe`, listens for `candle.update` / `candle.closed`, and returns `{ candles, status }`. Appends when a new bar starts, replaces the last bar when the same `time` ticks. Reconnect with exponential backoff starting at 1s, capped at 30s. Unsubscribes + closes on unmount or when `(symbol, timeframe)` change.
- **`frontend/src/components/symbol/LiveCandlestickChart.tsx`** — wraps the existing pure `<CandlestickChart>` with the hook and overlays a status pill in the top-right ("Connecting" / "Live" / "Reconnecting" / "Offline") so users know the feed's state.
- **Symbol page** — replaced `<CandlestickChart />` with `<LiveCandlestickChart symbol=ticker timeframe="1h" initial={candles} ... />`. SSR still passes the deterministic mock series so the chart is visible on first paint; the hook takes over once the socket is open.

## Verification

- `uvicorn app.main:app --port 8765` starts clean with the service logging `"market-data service started"`.
- `curl /api/v1/candles/AAPL?timeframe=1m&limit=3` → 200 with 3 candles of OHLCV at 60s intervals.
- Python websocket round-trip:
  ```
  subscribe AAPL/1m
    ← subscribed
    ← candle.update (price 264.65, volume 355)
    ← candle.update (price 264.85, volume 752)      # same bar, accumulating
    ← candle.update (price 265.16, volume 1249)
    ← candle.update (price 265.26, volume 1561)
  unsubscribe → unsubscribed
  ping → pong
  ```
  Bar `time` held constant across the four updates (same minute bucket), `high` ratcheted up, `volume` accumulated — aggregator correct.
- `pnpm build` on the frontend succeeds. Symbol route size: 55.6 → 56.8 kB (hook + status pill).

## Decisions & notes

- **Start with one provider — MockProvider.** It's a real implementation of the ABC, not a stub, and it produces plausibly-evolving ticks without any credentials or network. Adding Alpaca (or another vendor) in a later sprint is: write `providers/alpaca.py` implementing the same six methods, register the name in `_build_provider`, flip `MARKET_DATA_PROVIDER` in `.env`. No other file changes.
- **Reconnect lives in the service, not in each provider.** Providers just need to raise on stream failure. The outer loop in `MarketDataService._run` handles backoff and reconnection. Simpler providers and one consistent retry policy.
- **Bounded per-subscriber queues.** Without a cap, a tab that's throttled by the browser (backgrounded, slow network) would grow its queue until memory ran out. Capped at 100 events with a drop-oldest policy — chart accuracy degrades gracefully instead of everything falling over.
- **Candle-update vs candle-closed semantics.** The aggregator emits an in-progress `candle.update` on every tick (so the current bar animates), and a `candle.closed` the moment the bar flips. The hook treats them the same — it replaces the in-memory last bar — but downstream features (alerts, pattern engine) will often want to ignore anything that isn't `closed`.
- **Frontend keeps its mock SSR path.** Deleting `getMockSymbolDetail()` would mean the chart is empty until the hook wakes up (and CI/static-render would need a live backend). Keeping it means the page renders fast, offline-tolerant, and the live feed smoothly takes over once connected.
- **WS endpoint mounted at `/ws`, not under `/api/v1`.** Reverse proxies and nginx configs often terminate WebSockets differently from REST, and `NEXT_PUBLIC_WS_URL` was already defined as `ws://host/ws`. Keeping them on different paths avoids a prefix migration later.

## Not in scope (deferred)

- Alpaca / Polygon / Finnhub providers (one provider per the brief).
- Persisting ticks or candles to Postgres — the aggregator is in-process only.
- Auth on `/ws` — anyone who can reach the port can subscribe. Gating lands with server-side auth in CP-014.
- Pattern detection / strategy signal emission downstream of candles — CP-009 / CP-010.
- Smart resume after reconnect (backfilling missed bars).
