# CP-003 — Dashboard Build

**Date:** 2026-04-19
**Goal:** Build the Dashboard page with Market Overview, Watchlist, Top Signals, Alerts Feed, and AI Summary. Mock data first. Reusable components.

## Delivered

### Audit of existing work
All five widgets and the page shell were built during the original Figma port. CP-003's improvement surface was making the widgets *actually reusable* — they had their mock data baked in and accepted no props.

| Widget                    | File                                              | Status before CP-003 |
| ------------------------- | ------------------------------------------------- | -------------------- |
| Market Overview cards     | `components/dashboard/MarketOverview.tsx`         | Hardcoded data       |
| Watchlist widget          | `components/dashboard/Watchlist.tsx`              | Hardcoded data       |
| Top Signals Today         | `components/dashboard/TopSignals.tsx`             | Hardcoded data       |
| Alerts feed               | `components/dashboard/AlertsFeed.tsx`             | Hardcoded data       |
| AI Summary card           | `components/dashboard/AISummary.tsx`              | Hardcoded data       |
| Page composition          | `app/page.tsx`                                    | ✅                    |

### New in this sprint

- **`frontend/src/lib/mock-dashboard.ts`** — centralized, typed mock data. Exports:
  - Types: `MarketTile`, `SignalRow`, `WatchlistRow`, `DashboardAlert`, `DashboardAlertKind`, `AiSummary`, `AiSummaryTag`, `AiSummaryTone`, `AiSummarySegment`.
  - Constants: `mockMarkets`, `mockSignals`, `mockWatchlist`, `mockDashboardAlerts`, `mockAiSummary`.
- Every dashboard widget now accepts its data as an optional prop and falls back to the mock when none is supplied:
  - `<MarketOverview markets={…} />`
  - `<TopSignals signals={…} />`
  - `<Watchlist items={…} />`
  - `<AlertsFeed alerts={…} />`
  - `<AISummary summary={…} />`
- **AI Summary narrative became structured.** Instead of hardcoded JSX inside the component, the narrative is now an array of string segments and `{ticker}` refs. Tickers render as real `<Link>`s to `/symbol/[ticker]` — previously they were spans with no navigation.
- Empty-state copy added to Top Signals / Watchlist / Alerts Feed ("No active signals right now." etc.) so widgets degrade gracefully when the real feed returns zero rows.
- Watchlist header now shows a live item count next to the title.

### CP-008 seam

No changes required to the widget components.

## Verification
- `pnpm build` succeeds. Dashboard route bundle size: 1.42 → 1.43 kB (negligible).
- Mock data path still renders identical DOM to the previous hardcoded version for the four non-AISummary widgets.
- AI Summary picks up one intentional change: the SPY / QQQ ticker mentions are now real links. Visually still blue + medium weight.

## Decisions & notes
- **AI Summary narrative model.** Chose a `(string | {ticker: string})[]` segment array over a markdown-like format or raw JSX in the data module. Keeps the mock data module TS-only and lets a future CP-013 AI generator emit the same structured output.
- **Did not refactor the SectionCard usage.** AI Summary uses a custom blue/purple gradient card with decorative blur glows — intentionally different from the standard `SectionCard`. Extracting would flatten its identity.
- **Did not wire data fetching yet.** Backend lacks the endpoints (CP-008+). Defaulting to mock keeps the page useful standalone.

## Not in scope (deferred)

- Skeleton loading states — will add once fetching happens.
- AI narrative generation — CP-013.
